Die Erstellung einer Website beginnt in der Regel erst nach Abstimmung des Designs. Das Design wird dann in die Vorlagen des Bitrix Frameworks (sowie in die Komponentenvorlagen) integriert.

Die Einstellungen der Websitevorlagen überprüfen: "Einstellungen > Systemeinstellungen > Seiten > Seitenvorlagen". Für eine erfolgreiche Überprüfung sind ´Grundkenntnisse der Sprachen HTML und PHP erforderlich.

  1. Eine Vorlage muss einen verständlichen Namen haben, mit dem sie sich von den anderen Websitevorlagen unterscheidet. Der Name "Vorlage1" wäre in dieser Hinsicht ein eher schlechtes Beispiel.
  2. Zu Beginn der Vorlage muss im Element "<!DOCTYPE>" die HTML-Version angegeben werden, die den T3-Standards entspricht. Dieses Element ist sehr wichtig, denn es beeinflusst die visuelle Darstellung der Website und deren Arbeitsweise im Browser.
  3. Im Element "<HTML>" muss die Vorlagensprache angegeben werden. Die Sprache wird dabei aus den Websiteeinstellungen genommen. Ein Beispiel: lt;html xmlns="http://www.w3.org/1999/xhtml" xml:lang="<?=LANGUAGE_ID?>" lang="<?=LANGUAGE_ID?>">
  4. Wenn die Angabe der Seitenüberschriften automatisch erfolgt, können die Namen der Web-Seiten und die Browser-Überschriften einfach über die Benutzeroberfläche geändert werden. <title><?$APPLICATION->ShowTitle()?></title>?>
  5. Meta-Tags der Web-Seite (keywords, description, content-type, robots), sowie Stile und Skripte sollten ebenfalls automatisch angezeigt werden: <?$APPLICATION->ShowHead();?>
  6. Das Website-Icon sollte festgelegt werden: <link rel="shortcut icon" type="image/x-icon" href="<?=SITE_TEMPLATE_PATH?>/favicon.ico" />
  7. Das administrative Menü, über welches die Website-Steuerung erfolgt, sollte automatisch angezeigt werden: <body><div id="panel"><?$APPLICATION->ShowPanel();?></div>
  8. Die Telefonnummer, die Copyright-Jahresangabe im Footer der Website sowie andere eingebettete Bereiche können über das Pop-up oder aber über den administrativen Bereich bearbeitet werden. Dazu sind keine Programmierkenntnisse erforderlich.
  9. In der Website-Vorlage sind Werbebereiche (Werbebanner) definiert: Diese können im Bereich Ansicht (beim eingeschalteten Bearbeitungsmodus) oder im Bereich Administration (Services > Werbung > Banner) bearbeitet werden.
  10. Alle "komplizierten" Web-Design-Elemente müssen sich im Header bzw. Footer der Website befinden. Im Arbeitsbereich (WORK_AREA) sollten sich lediglich die "einfacheren" Gestaltungselemente befinden. So wird das Risiko vermieden, dass der Content-Manager die Gestaltung der Website durch seine Arbeit im visuellen Editor aus Versehen "beschädigt". Diese Vorsichtsmaßnahme ist besonders wichtig für die Startseite.
  11. Die Website- und Vorlagenstile (welche auf den gleichnamigen Registerkarten angezeigt werden) müssen ordentlich eingerichtet und ausführlich dokumentiert werden.
